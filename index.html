部署方法

GitHub Pages

最简单的方式，适合静态网站，无需额外配置

  1. 创建仓库 - 确保您的仓库是公开的（或使用GitHub Pro账户）
  2. 添加网站文件 - 在仓库中添加index.html作为入口文件
  3. 启用GitHub Pages
    # 在仓库设置中：
    Settings → Pages → Branch: main
    → Folder: / (root) → Save
  4. 访问网站 - 通过 https://<username>.github.io/<repository> 访问

优点

  • 完全免费
  • 无需服务器配置
  • 与GitHub无缝集成
  • 自动HTTPS支持

限制

  • 仅支持静态内容
  • 构建时间限制
  • 流量限制（100GB/月）

自定义部署

灵活的方式，适合动态网站和高级需求

  1. 选择托管平台 - Vercel, Netlify, AWS, Heroku等
  2. 连接GitHub仓库 - 授权平台访问您的仓库
  3. 配置构建设置
    # 示例：Netlify部署命令
    Build command: npm run build
    Publish directory: dist/
  4. 自定义域名 - 添加您自己的域名（可选）
  5. 部署 - 每次git push后自动部署

优点

  • 支持动态内容
  • 更高级的配置选项
  • 全球CDN加速
  • 免费计划可用

限制

  • 需要更多配置
  • 免费计划有资源限制
  • 学习曲线较陡峭

HTML网站示例

基本HTML网站结构

为您的GitHub仓库创建一个简单的网站：

<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>我的笔记网站</title>
  <style>
   /* 添加您的CSS样式 */
  </style>
</head>
<body>
  <header>
   <h1>我的笔记</h1>
  </header>

  <main>
   <section id="notes">
    <!-- 动态加载笔记内容 -->
   </section>
  </main>

  <footer>
   <p>© 2023 我的笔记网站</p>
  </footer>

  <script>
   // 添加JavaScript功能
  </script>
</body>
</html>

部署说明： 将此文件保存为index.html放在仓库根目录，然后启用GitHub Pages。
